Для ручки get_users_handler решил добавить разные фильтры для отбора пользователей.

Из-за передачи фильтров в Body, оставил метод, как Post. Фильтрация работает по следующей схеме:
- Если есть id пользователя - возвращаем одного пользователя, и завершаем запрос
- Если есть login пользователя - возвращаем одного пользователя, и завершаем запрос
- В других случаях делаем фильтрацию по всем остальным параметрам
- Дополнительно есть фичефлаг "only_available", который фильтрует занятых пользователей

От себя еще добавил ручку create_users, через которую можно засылать сразу нескольких пользователей.

Развертывание через docker-compose, миграции сделаны через alembic, сервис и бд работают в асинхронном режиме.
